Capítulos 7 a 12.

HTML é o esqueleto. CSS é a pele.

Separar estrutura de apresentação.

Três formas — uma recomendada

<link rel="stylesheet" href="css/style.css">  <!-- ✔️ externo -->

<style> h1 { color: navy } </style>        <!-- interno -->

<p style="color: navy">                     <!-- inline (evite) -->

Externo = cacheável, reutilizável, separação de responsabilidades.

Anatomia de uma regra

seletor {
  propriedade: valor;
}

h1 {
  color: #12243A;
  font-size: 2rem;
}

Seletor escolhe quem; declarações dizem como.

Cor e fonte

Cada declaração muda um aspecto visual do mesmo parágrafo.

Borda, padding e margin

Padding = espaço interno (dentro da borda); margin = espaço externo (entre caixas). O fundo azul mostra até onde vai o padding.

box-sizing: border-box faz width incluir padding e border — sempre use.
O reset universal *{box-sizing:border-box} economiza horas de frustração. Explique por que 300px + padding "estoura" sem ele.

Veja o box model

O resultado é renderizado pelo próprio código à esquerda. Troque box-sizing para content-box e a caixa "estoura" (largura passa a somar padding + borda).

Unidade certa, no lugar certo

  • rem — fontes e espaçamentos globais (respeita o usuário)
  • em — proporcional ao contexto local
  • % / fr — larguras fluidas
  • vw/vh — relativo à janela

Evite px em font-size: quebra o zoom de acessibilidade.

Pare de lutar com float.

Flexbox e Grid resolvem layout de verdade.

Flexbox: uma dimensão

.container {
  display: flex;
  justify-content: center;  /* eixo principal */
  align-items: center;      /* eixo cruzado */
  gap: 1rem;
}

Dois eixos: principal (direção do flex) e cruzado (perpendicular).

flex-direction troca qual eixo é o principal — é a fonte de toda confusão. gap substitui margens entre itens.

O centralizar que todo mundo procura

display:flex; justify-content:center; align-items:center — centro perfeito, horizontal e vertical.

Centralizar de verdade

O resultado à direita é renderizado pelo próprio código à esquerda.

Grid: duas dimensões

.galeria {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1rem;
}

fr = fração do espaço livre. repeat() evita repetição.

Grid é para as duas dimensões ao mesmo tempo. fr distribui o que sobra.

Áreas nomeadas: layout como desenho

grid-template-areas:
  "cabecalho cabecalho"
  "menu      conteudo"
  "rodape    rodape";

O código é o wireframe.

Responsivo sem media query

grid-template-columns:
  repeat(auto-fit, minmax(220px, 1fr));

Os cards se reorganizam sozinhos (padrão RAM).

Uma galeria com Grid

Troque para repeat(auto-fit, minmax(80px, 1fr)) e redimensione a janela para ver os itens se reorganizarem sozinhos (padrão RAM).

Flexbox ou Grid?

Flexbox

Uma dimensão: uma linha ou uma coluna. Menu, barra, lista de tags.

Grid

Duas dimensões: linhas e colunas. Layout de página, galeria.

Não competem — combine: Grid no macro, Flex no micro.

Uma página, todas as telas.

Mobile-first.

Responsivo = mobile-first

/* base: mobile */
.grid { grid-template-columns: 1fr; }

/* a partir de 768px: sobe para 3 colunas */
@media (min-width: 768px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}
Sem <meta name="viewport"> no HTML, nada é responsivo.
Breakpoint segue onde o layout quebra, não a largura de um iPhone específico.

Tipografia fluida

h1 { font-size: clamp(1.75rem, 4vw, 3rem); }
/*              mínimo    ideal  máximo */

Escala suave entre dois limites — sem breakpoint.

Design System: decisões, não acasos

:root {
  --azul-600: #2563eb;              /* primitivo */
  --cor-interativa: var(--azul-600); /* semântico */
}
.btn { background: var(--cor-interativa); }

Token primitivo → token semântico → componente.

Muda a cor interativa em UM lugar e o site inteiro acompanha. É o coração do Design System.

Dark mode = trocar tokens

:root            { --bg: white; --texto: #111; }
[data-theme=dark]{ --bg: #111;  --texto: #eee; }

Um seletor troca o tema inteiro. Nenhum componente muda.

Um componente com tokens

Passe o mouse no card durante a apresentação — o iframe é interativo e a sombra do :hover aparece de verdade. Note as variáveis (tokens) --cor e --raio em uso.

Tailwind: utility-first

CSS tradicional

<button class="btn">
  OK
</button>
/* + regra .btn no CSS */

Tailwind

<button class="px-4 py-2
  bg-blue-600 text-white
  rounded">OK</button>

Classes utilitárias = tokens de um Design System prontos.

É o cap. 11 automatizado: a escala do Tailwind é um Design System. Trade-off: HTML "verboso" vs. zero troca de contexto e consistência por convenção.

Responsivo e estados por prefixo

<div class="flex-col md:flex-row hover:bg-blue-700">

md: = a partir do breakpoint; hover: = no estado.

Resumo

  • Seletor + especificidade + cascata decidem o estilo.
  • Tudo é uma caixa (use border-box).
  • Flexbox (1D) e Grid (2D) fazem o layout.
  • Responsivo é mobile-first + clamp().
  • Tokens e Tailwind escalam o design.
