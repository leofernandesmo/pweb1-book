MPA × SPA

MPA

Cada navegação = novo request e recarga da página inteira.

SPA

Um HTML; o JS troca o conteúdo com base na URL, sem recarregar.

SPA = navegação fluida e estado preservado. Custo: SEO e carga inicial exigem atenção.
Sem React/Vue — só JavaScript moderno. O objetivo é entender o que os frameworks automatizam.

Separe em camadas

┌─── UI ────────── pages/, components/ (DOM, eventos)
│        ↓ chama
├─── State ─────── store.js (estado + notifica a UI)
│        ↓ chama
└─── Service ───── services/ (fetch, cache, erros)
Regra de ouro: a UI nunca faz fetch; o Service nunca toca no DOM.
Essa separação deixa cada camada testável e substituível. É o hábito que separa script de projeto.

Módulos ES6

// services/api.js
export async function buscar(url) { /* ... */ }

// app.js
import { buscar } from './services/api.js';

Cada arquivo tem escopo próprio e exporta só o que é público.

O fluxo de uma SPA

URL (#/produtos)→ router→ service→ render no DOM

Hash routing (#/) — simples e perfeito para GitHub Pages.

A síntese do curso.

Uma SPA que consome uma API pública

  • Listagem com filtro e busca
  • Tela de detalhe
  • Formulário validado
  • Navegação sem recarregar

  • Persistência (localStorage)

8 funcionalidades obrigatórias — tudo que você aprendeu, em um app.
